Deployment Guide — GWM Daily Sales Reporting Tool

Follow these five steps to go live: create your Google Sheet, deploy the Apps Script backend, host on GitHub Pages, connect the dashboard, then optionally wire up Power BI. Each step takes 5–10 minutes.

1
Create the Google Sheet
▾
  1. Go to sheets.google.com and create a new blank spreadsheet.
  2. Name it: GWM Daily Sales – Southern Region
  3. Copy the URL — find the long ID between /d/ and /edit. This is your SHEET_ID.
  4. The Apps Script will auto-create or safely upgrade the correct tab and header row on first run. You do not need to add headers manually.

2
Deploy the Google Apps Script Web App
▾
  1. Open your Google Sheet → click Extensions → Apps Script.
  2. Delete any existing code in Code.gs.
  3. Paste the entire contents of code.gs (from this project).
  4. Find line 10 and replace YOUR_GOOGLE_SHEET_ID_HERE with your actual Sheet ID: 
    const SHEET_ID = '1BxiM..........XYZ';
  5. Optionally set an ACCESS_CODE on line 12 for basic link protection.
  6. Click Save (Ctrl+S / Cmd+S).
  7. Click Deploy → New deployment.
  8. Type: Web app
  9. Execute as: Me
  10. Who has access: Anyone (this allows dealer phones to POST without login)
  11. Click Deploy → authorise when prompted → copy the Web app URL.
✓ Test it: In the Apps Script editor, select the function testSetup and click Run. Check the Logs — it should say "Sheet ready".
Every time you edit code.gs you must Deploy → Manage deployments → Edit → New version to publish changes. Existing URL stays the same.
3
Host on GitHub Pages
▾
  1. Create a new repository at github.com — name it e.g. gwm-sales-reporting.
  2. Upload these four files to the root of the repository:
    • index.html
    • dashboard.html
    • styles.css
    • app.js
  3. Go to repository Settings → Pages.
  4. Source: Deploy from a branch → Branch: main → folder: / (root).
  5. Click Save. GitHub will provide a URL like:
    https://your-org.github.io/gwm-sales-reporting/
  6. The site is live in 1–2 minutes. Share index.html with dealers and dashboard.html with internal GWM staff.
Optional: Set a custom domain (e.g. gwm-reports.yourdomain.com.au) via the GitHub Pages custom domain setting + a CNAME DNS record.

Paste the Apps Script URL into app.js

Open app.js, find line 9, and replace the empty string with your Web App URL: 

const GWM_CONFIG = {
  scriptUrl: 'https://script.google.com/macros/s/XXXXX/exec',
  region: 'Southern Region',
  cutoffHour: 10,
};

Commit and push. Alternatively, paste the URL directly in the dashboard's ⚙ Configure button (bottom-right) — this stores it in the browser's localStorage so you don't need to rebuild.

4
Verify the Dashboard
▾
  1. Open dashboard.html in your browser.
  2. Click the ⚙ button (bottom-right) and paste your Apps Script URL.
  3. Click Save & Reload — the demo ribbon should disappear and live data should load.
  4. Submit a test report from index.html using any dealer.
  5. Return to the dashboard and click ↻ Refresh — confirm the dealer now shows as "Submitted" in the Dealer Compliance table.
  6. Check your Google Sheet — you should see 19 new rows (one per model bucket).
Late flag test: Temporarily change cutoffHour: 10 to cutoffHour: 0 in app.js to force all submissions to be flagged late. Revert after testing.
5
Connect to Power BI
▾

Option A — Google Sheets Connector (live refresh)

  1. Open Power BI Desktop → Get Data → Web.
  2. Use the Google Sheets direct CSV export URL format:
    https://docs.google.com/spreadsheets/d/YOUR_SHEET_ID/gviz/tq?tqx=out:csv&sheet=submissions
  3. Power BI will load a flat CSV. Set the first row as headers.
  4. Set column types:
    • submitted_at → Date/Time
    • report_date → Date
    • is_late → True/False
    • enquiry, test_drives, new_sold, fleet_5_plus, demo_sold, forecast → Whole Number
  5. Click Close & Apply.
  6. Set refresh schedule in Power BI Service: Daily or Hourly.
The Google Sheet must be shared as "Anyone with the link can view" for Power BI to access it without OAuth.

Option B — CSV Export (manual / scheduled)

  1. In the dashboard, set your date filter and click ⬇ Export CSV.
  2. In Power BI Desktop → Get Data → Text/CSV → select your downloaded file.
  3. For scheduled automation: use a Power Automate flow to call your Apps Script GET endpoint daily and save the result as a CSV to SharePoint or OneDrive, then connect Power BI to that file.

Recommended Power BI Measures

Compliance % =
DIVIDE(
  DISTINCTCOUNT(submissions[dealer_code]),
  30   -- total dealer count
) * 100

Conversion Rate =
DIVIDE(
  SUM(submissions[new_sold]),
  SUM(submissions[enquiry])
) * 100

Late Submissions =
CALCULATE(
  DISTINCTCOUNT(submissions[dealer_code]),
  submissions[is_late] = TRUE
)

Suggested Visuals

  • Card visuals — Submissions Today, Compliance %, Total Enquiries, New Sold
  • Bar chart — New Sold by Model Bucket (filter to current month)
  • Line chart — Daily Enquiry Trend (date on X axis)
  • Matrix — Dealer × Model Bucket (values: New Sold)
  • Donut — Direction Split (Up / Flat / Down)
  • Table — Dealer Compliance with conditional formatting on is_late
